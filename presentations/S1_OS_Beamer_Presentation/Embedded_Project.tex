\begin{frame}
\frametitle{Was ist YAML} 
\begin{itemize}
\setlength{\itemsep}{0.6cm}
\item \glqq YAML Ain't Markup Language \grqq\\ 

\item vereinfachte Auszeichnungssprache\\
mensch- und maschinenlesbar\\
Gliederung und Formatierung von Text und Daten\\

\item zur Datenserialisierung\\
\item ursprünglich an XML angelehnt\\
\item Datenstrukturen von Perl, Python und C\\
\item entwickelt von Clark Evans (2001)

\subsection{Vergleich mit JSON}
\begin{frame}
\frametitle{Vergleich mit JSON}
\begin{itemize}
\item[\textbullet] JSON ist Untermenge von YAML\\
\item[\textbullet] JSON Dokument ist gleichzeitig YAML 1.2 Dokument\\ (V1.1 nicht!)
\item[\textbullet]JSON Designziele: Einfachheit, Universalität
\item[\textbullet]YAML Designziele: Lesbarkeit durch Mensch, Unterstützt serielle Übertragung von Datenstrukturen 
\item[\textbullet] Was YAML erlaubt und JSON nicht: ~\cite{cow}
\begin{itemize}
\item[\textbullet] Kommentare
\item[\textbullet] Tags für Metadaten
\item[\textbullet] Anchors
\item[\textbullet] Dokument Start- und Endmarkierungen
\end{itemize}
\end{itemize}
\end{frame}

\subsection{Designkriterien des Protokolls}
\begin{frame}
\frametitle{Designkriterien des Protokolls} 
Designziele (nach Priorität): 
\begin{itemize}
\item[\textbullet] Lesbarkeit für den Menschen\\
\item[\textbullet] Defaultparameter sind vorhanden\\
\item[\textbullet] Reihenfolge der Parameter beliebig\\
\item[\textbullet] Wenig Overhead (Geschwindigkeit)\\
\end{itemize} 
\end{frame}

\subsection{Beschreibung und Abhängigkeiten}
\begin{frame}
\frametitle{Beschreibung und Abhängigkeiten}
Parametername hätte auch weggelassen werden können.\\ 
(Verwendung von Array statt Assoziativen Listen)\\
\vspace{0.3cm}
Wir haben uns dagegen entschieden weil:
\begin{itemize}
\item[\textbullet] Lesbarkeit verschlechtert wird\\ %\textendash
\item[\textbullet] Reihenfolge der Parameter nicht mehr beliebig wäre
\end{itemize}
\vspace{0.3cm}
Nachteile dieser Entscheidung:
\begin{itemize}
\item[\textbullet] Overhead (Geschwindigkeit)
\end{itemize}
\end{frame}

\section{Bewertung}
\subsection{Vorteile}

\begin{frame}
\frametitle{Vorteile} 
YAML:
\begin{itemize}
\setlength{\itemsep}{0.3cm}
\item[++] Leicht vom Mensch Schreiben, Lesen (vgl. XML)
\item[++] Weiterverarbeitung der Daten in anderen Sprachen
\item[+] Hinzufügen von Tags erlaubt
\item[+] Sprache ist leicht zu Verwenden
\end{itemize}

\vspace{0.3cm}
Entwickeltes Protokoll:
\begin{itemize}
\setlength{\itemsep}{0.3cm}
\item[++] Leicht vom Mensch Schreiben, Lesen (vgl. XML)
\item[+] Defaultparameter sind vorhanden (nicht alle Senden)
\item[+] Reihenfolge der Parameter ist beliebig
\end{itemize}
\end{frame}

\subsection{Nachteile}
\begin{frame}
\frametitle{Nachteile}
YAML:
\begin{itemize}
\setlength{\itemsep}{0.3cm}
\item[-- --] Geschwindigkeit (vgl. JSON)
\item[-- --] Verbreitung und Bekanntheit
\item[\textendash] Wird immer noch weiterentwickelt (Ausgereiftheit)
\end{itemize}
\vspace{0.3cm}

Entwickeltes Protokoll:
\begin{itemize}
\setlength{\itemsep}{0.3cm}
%\item[-- --] Leicht vom Mensch Schreiben, Lesen (vgl. XML)
\item[\textendash] Vermeidbarer Overhead (Siehe Abhängigkeiten)
\item[\textendash] Sender und Empfänger brauchen Befehlsliste
\end{itemize}
\end{frame}
